# NOC Toolkit — Краткий справочник

**Версия:** 0.6.0 | **Запуск:** `python3 noc-toolkit.py`

---

## 1. PagerDuty-Jira Tool

Синхронизация PagerDuty инцидентов с Jira тикетами.

- Получает ваши PD инциденты и автоматически находит связанные Jira тикеты (из названия + комментариев)
- Публикует обновления Jira статусов как PD комментарии
- Режим `--snooze`: авто-снуз после обновления (настраиваемый таймер)
- Распознаёт ключевые слова "ignore" / "disabled" → короткий комментарий + снуз
- Защита от дублирования комментариев (12-часовое окно)

**Режимы:** `--update` (только статус) | `--snooze` (статус + снуз)

---

## 2. PagerDuty Job Extractor

Извлечение имён упавших джобов из смерженных PD инцидентов.

- Принимает URL или ID инцидента
- Парсит смерженные алерты и извлекает имена `jb_*` джобов
- Выводит чистый список для дальнейшего разбора

---

## 3. PagerDuty Monitor

Авто-подтверждение triggered инцидентов.

- Подтверждает все triggered инциденты, назначенные на вас
- Оставляет рандомизированный человекоподобный комментарий (13 фраз + 10 с опечатками)
- **Тихий ack** (без комментария) для «Missing» инцидентов load-status:
  - Missing AUS & NZL, MSP Export, CANADA, Central, East, International
- Интерактивное меню длительности (1ч / 2ч / 4ч / 8ч / 12ч / произвольное)
- `--once` для однократной проверки, `--details` для подробного вывода

---

## 4. PagerDuty Incident Merge

Поиск и объединение связанных PD инцидентов по имени джоба.

- **Сценарий A** — Дубли за один день: одинаковый джоб, одна дата → мерж
- **Сценарий B** — Кросс-дата с Jira: одинаковый джоб, разные даты, есть Jira тикет → мерж
- **Сценарий C** — Массовый сбой: DSSD инцидент с 10+ смерженными алертами → объединение связанных джобов
- **Сценарий D** — RDS Exports: объединение `RDS export <job> failed` в `RDS exports - failed to start`
- Интерактивное подтверждение для каждой группы и каждого инцидента
- Skip-список запоминается между запусками (интерактивная очистка при старте)

**Флаги:** `--dry-run` | `--verbose` | `--show-skips`

---

## 5. Data Freshness Checker

Ежедневный отчёт свежести данных DACSCAN через Databricks SQL.

- Основной отчёт из 15 строк (DACSCAN, AGG, AUDIT, SUMMARY, BI-LOADER таблицы)
- Автоматические гранулярные проверки для задержанных таблиц (по хостам для DACSCAN, max(update_ts) для агрегатов)
- Обратный отсчёт до SLA (дедлайн 5:30 PM UTC)
- HTML-отчёт с цветовой индикацией — открыть в браузере для скриншотов в Slack

**Флаги:** `--report` (HTML) | `--check-all` | `--dry-run` | `--format csv/json`

---

## 6. NOC Report Assistant

Синхронизация Jira статусов в End-of-Shift Excel отчёт.

- **Sync statuses** — обновляет Jira статус (столбец E) для всех существующих тикетов
- **Add row** — вставляет новый тикет в секцию "Things to monitor" с Jira + Slack ссылками
- Работает с листами Night-Shift-NEW и Day-Shift-NEW
- Сохраняет всё форматирование Excel, объединения ячеек, гиперссылки

**Флаги:** `--dry-run` | `--file PATH` (по умолчанию: `~/Downloads/NOC endshift report.xlsx`)

---

## 7. PD Escalation Tool

Автоматизация воркфлоу эскалации после создания DSSD.

- Автоопределение DRGN тикета из PD инцидента (через Jira integration field)
- Создаёт Jira линк: DRGN "is blocked by" DSSD
- Переводит DRGN в статус "Escalated"
- Публикует PD заметку с описанием эскалации
- Выводит готовый Slack шаблон для #cds-ops-24x7-int

**Использование:** `--pd <incident_id> --dssd DSSD-XXXXX [--drgn DRGN-XXXXX] [--dry-run]`

## Переменные окружения

Все инструменты используют общий файл `.env` в корне toolkit:

| Переменная | Используется | Описание |
|------------|--------------|----------|
| `PAGERDUTY_API_TOKEN` | 1, 2, 3, 4, 7 | PagerDuty API токен (с правами записи) |
| `JIRA_SERVER_URL` | 1, 4, 6, 7 | URL Jira сервера |
| `JIRA_PERSONAL_ACCESS_TOKEN` | 1, 4, 6, 7 | Jira PAT (Bearer авторизация) |
| `DATABRICKS_HOST` | 5 | URL Databricks workspace |
| `DATABRICKS_TOKEN` | 5 | Databricks access token |
| `DATABRICKS_WAREHOUSE_ID` | 5 | ID SQL warehouse |

---

## Советы

- Все инструменты поддерживают `--dry-run` для безопасного предпросмотра
- Запустите toolkit и нажмите номер инструмента — аргументы CLI не нужны
- Нажмите **Ctrl+C** чтобы прервать любой инструмент и вернуться в меню
- Файл `.env` загружается автоматически при запуске
