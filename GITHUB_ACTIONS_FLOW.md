## 🎯 Два workflow на выбор

### Workflow 1: Build Windows EXE

```
Push → GitHub → Windows VM → Build .exe → Artifact
  ↓
5-10 минут
  ↓
Скачать готовый .exe
```

**Использовать когда:**
- Нужен только Windows .exe
- Быстрая сборка
- Тестирование изменений

---

### Workflow 2: Build Multi-Platform Release

```
Manual trigger → GitHub → 3 VMs параллельно:
  ├── Windows VM → NOC-Toolkit.exe
  ├── macOS VM   → NOC-Toolkit (Mac)
  └── Linux VM   → NOC-Toolkit (Linux)
       ↓
   10-15 минут
       ↓
   3 артефакта + checksums + summary
```

**Использовать когда:**
- Официальный релиз
- Нужна поддержка всех платформ
- Создание дистрибутивов

## 🔀 Варианты триггеров

### Автоматический (build-windows.yml)

```
Code change → git push → main branch
                   ↓
        Автоматически запускается сборка
                   ↓
           Артефакт готов через 10 мин
```

**Плюсы:**
- Автоматизация
- Не нужно помнить запустить
- Всегда актуальный .exe

**Минусы:**
- Расход минут при каждом push
- Может быть избыточно для мелких изменений

---

### Ручной (workflow_dispatch)

```
GitHub Actions → "Run workflow" → Click
                        ↓
              Сборка запускается
                        ↓
             Артефакт готов через 10 мин
```

**Плюсы:**
- Контролируешь когда собирать
- Экономия минут
- Только когда нужно

**Минусы:**
- Нужно помнить запустить

## 📊 Сравнение методов сборки

| Метод | Где собирается | Время | Платформа | Автоматизация |
|-------|----------------|-------|-----------|---------------|
| **GitHub Actions** | Облако GitHub | 5-10 мин | Win/Mac/Linux | ✅ |
| **Mac PyInstaller** | Твой Mac | 2-5 мин | только Mac | ❌ |
| **Товарищ Windows** | Windows 11 | 3-5 мин | только Windows | ❌ |
| **Wine на Mac** | Твой Mac (эмуляция) | 10-30 мин | Windows (не гарантия) | ❌ |

**Вывод:** GitHub Actions - оптимальный выбор! ⭐

## 🎯 Рекомендованный flow

### Разработка:

```
1. Редактируешь код на Mac
2. Тестируешь локально: python3 noc-toolkit.py
3. Commit: git commit -m "Add feature"
4. Push: git push
5. GitHub автоматически собирает .exe
6. Скачиваешь из Artifacts
7. Тестируешь .exe (или передаёшь товарищу на тест)
```

### Релиз:

```
1. Обновить VERSION.md
2. Commit: git commit -m "Release v1.0.0"
3. Push: git push
4. GitHub Actions → "Build Multi-Platform Release"
5. Ввести версию: v1.0.0
6. Подождать 15 минут
7. Скачать все 3 артефакта
8. Создать GitHub Release с артефактами
9. Опубликовать для команды
```
